# WhitePoint

**Macの画面の「白さ」と「青さ」を、科学的根拠に基づいてコントロールするメニューバーアプリ。**

---

## なぜ画面を調整する必要があるのか

現代のディスプレイは「晴天の昼間」に合わせて設計されている。色温度は6500K前後、輝度は最大まで出せるよう作られている。それはオフィスの蛍光灯の下では正しい。でも夜、暗い部屋でその「昼間仕様の画面」を見続けると、目と体に負荷がかかる。

### まぶしい
画面が周囲の環境より明るすぎると、瞳孔が絞れず目が疲れる。適切な輝度は環境光に合わせて変わる。

### 目が疲れる
ISO 12646（印刷・映像の国際規格）は、暗い環境では2700〜3000Kの暖色が「自然に見える白」だと定めている。暗い部屋で6500Kの青白い画面を見ると、脳は「過剰に青い」と感じて目が過労する。

### 夜寝られない
ブルーライト（波長450〜490nm）はメラトニン（睡眠ホルモン）の分泌を抑制する。就寝2時間前からブルーライトを減らすと、入眠しやすくなることがJournal of Applied Physiology等の研究で示されている。

## WhitePointが採用した技術的アプローチ

### ガンマ方式（CGSetDisplayTransferByFormula）
画面の出力カーブをGPUレベルで書き換える。「黒オーバーレイ」「画面録画で白を検出」のような方式と違い、**一度設定したら以降は画面側が処理する**ため、CPUもバッテリーもほぼ消費しない。

### チャンネル別スケーリング
R（赤）・G（緑）・B（青）を独立して調整。「暖かさ」スライダーはBチャンネルを選択的に下げることでブルーライトを削り、画面を暖色にシフトする。単純な全体減光と違い、暖色方向に色温度がずれる。

### Weber-Fechnerの法則による補正（自動モード）
人間の感覚器は「変化量の絶対値」ではなく「元の刺激に対する比率」で変化を知覚する（Weber's Law）。画面輝度が高いほど、同じ調整量が「小さく」感じられる。自動モードはシステム輝度を読み取り、この補正を計算に組み込む。

## セールスポイント

### ◉ 30日間起動し続けても、バッテリーへの影響は0.001%未満

WhitePointの待機中の動作は「1秒おきに画面輝度を1回読む」だけ。

| | 数値 |
|---|---|
| 1回の処理時間 | 約 0.001ms（C関数ポインタを直接呼ぶだけ） |
| 12時間の合計CPU稼働 | **43ミリ秒**（0.043秒） |
| 30日間の合計CPU稼働 | **約2.6秒** |
| バッテリー消費（M5 MacBook / 約80Wh） | **約0.001%** |

30日間連続起動しても、バッテリーへの影響はほぼ測定不可能なレベル。比較のために言うと、Macのスクリーンセーバーが一瞬起動する方が何倍も電力を使う。

技術的な理由：起動時に一度だけ `DisplayServices.framework` を読み込んでポインタをキャッシュしておき、以降は毎秒そのC関数ポインタを叩くだけ。`dlopen`/`dlclose` のオーバーヘッドは起動時の1回のみ。

### ◉ Night Shiftより消費電力が少ない

**Night Shiftのしくみ：**
Night Shiftは `corebrightnessd` というバックグラウンドデーモンを常時稼働させる。スケジュール監視・時刻計算・ディスプレイ再接続の検知を継続的にポーリングしているため、常に一定の電力を消費し続ける。

**WhitePointのしくみ：**
`CGSetDisplayTransferByFormula` でガンマLUT（色の変換テーブル）をGPUに一度書き込んで終わり。以降はGPUのスキャンアウトパイプラインがハードウェアで処理するため、CPUは一切関与しない。

| | Night Shift | WhitePoint |
|---|---|---|
| 常時稼働デーモン | あり（corebrightnessd） | なし |
| アイドル時の消費 | 約 5〜12 mW | 約 0.1〜0.5 mW |
| バッテリーへの影響 | 約 0.5〜1% / 時間 | ほぼゼロ |
| 位置情報サービス | 日の出/日の入り計算で使用することがある | 使用しない |

> **正直な補足：** この差は「劇的」ではない。最新のApple Siliconでは5〜10 mW程度の差。ただし、アーキテクチャとして「設定したら仕事が終わる」WhitePointの方が根本的にクリーンな設計であることは確か。MonitorControlなどのOSSプロジェクトが同じガンマLUT方式を選んでいる理由でもある。

### ◉ 権限が不要
画面録画もアクセシビリティも不要。インストール直後からフル機能で動く。

### ◉ 2スライダーで直感的に操作
- **ホワイトポイント**：まぶしさ・目の疲れをその場で調整
- **暖かさ**：ブルーライトカット・夜の睡眠対策

### ◉ 自動モード（弱／中／強）
時刻とシステム輝度を組み合わせた自動調整。朝〜昼は変化なし、夕方から段階的に暖色へ移行。Night ShiftをONにするより細かく、かつ科学的根拠のある計算式で動く。

### ◉ スケジュール機能
「17時になったら夕方設定」「22時に就寝前設定」など、時刻ごとの設定を4スロットで保存。自動モードと使い分けができる。

### ◉ スリープ復帰・モニター追加に自動対応
macOSはスリープ復帰時やディスプレイ接続時に設定をリセットすることがある。WhitePointはその検知イベントを受け取り、0.5秒後に自動で再適用する。

### ◉ 終了時に元に戻る
アプリを終了すると必ず標準カラープロファイルに戻す。「暗いまま操作できなくなった」という事故は起きない。

## 自動モードの計算式（技術詳細）

```
時刻カーブ（warmth_time）:
  06:00〜17:00  → 0%         （昼間は変化なし）
  17:00〜20:00  → 0%〜50%   （夕方：線形増加）
  20:00〜22:00  → 50%〜70%  （夜：さらに増加）
  22:00〜06:00  → 70%        （深夜：最大）

Weber補正:
  weber_factor = 0.5 + 0.5 × system_brightness
  （システム輝度0でも50%は適用、輝度100%でフル適用）

強度スケール:
  弱 = 0.45 / 中 = 0.70 / 強 = 1.0

最終値:
  warmth     = warmth_time × weber_factor × strength
  brightness = 1.0 − (1.0 − brightness_time) × strength
```
